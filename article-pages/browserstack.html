Screenshots By Browserstack

Screenshots product by BrowserStack is used to test cross-browser compatibility of websites. Using this product users can generate screenshots across combinations of 2000+ devices and browsers, these screenshots are used to compare designs.

Problem statement

  • Highlighting the length and breadth of the OS and browser versions offered by BrowserStack.

  • Finding specific browser and OS combinations isnâ€™t intuitive.

  • Interaction for selecting multiple combinations to generate multiple screenshots at once may be time-consuming.

Design explorations

Started by conducting quick research on Browserstack Screenshots. The Goal was to find a pattern of highlighting the length and breadth of the OS and browser versions offered by BrowserStack.

In the design iterations, I tried to cover up the following points -

  • Allowing the user to select all the versions of a browser for a specific OS.

  • Letting the user select the length and breadth of OS and Browser versions.

Iteration 1 (Desktop view)

Used a toggle view with a set of secondary search options like - OS selection, Browser selection, and resolution selection.

All the 3 secondary search options will open in the tooltip format containing the options of the user to select.

The user can cancel all his selections in the secondary search followed by a segment.

For the solution, I have used only 2 OS and the browsers associated with them. The data used here is for design purpose only. I can maybe incorrect with the browser and resolution numbers.

There is a slight change in the search input. I have moved the recent out of the input along with the capture button. Just a slight design change as I feel confused where the generate or capture button is.

The toggle helps to switch from desktops to mobile devices and vice versa.

In the browser versions, I am not completely showing all the versions by showing the user only the latest ones. Most of the users will be there to check their product on the latest and some of the second preferred versions. In case a user wants to see all the versions he can tap on the More link which will appear all the versions.

To select all the version of a browser on a specific OS. User can simply tap on select all to simply select all the versions at once.

When the user taps on generate screenshots button or capture button he will be redirected to the Browserstack Screenshots results page. Nothing changed here.

The setting link below the generate screenshots button acts as a panel. A small panel will come up for further advanced options.

Iteration 1 (Web Full view)



These screens show the behavior when the user taps on the more link and while clicking on the select all.

Iteration 1 (Mobile toggle view)

Users can easily select the mobile or web with the help of the toggle. The toggled view helps to separate the web segments from the mobile for easy access.

Iteration 2 (Using custom filters)

Another way to round up things is to use a filter for the OS, Browser, and Resolution.

Iteration 2 (Filter's Open)

When you click on the filter icon all the existing options appear here instead of spreading or aligning them throughout the page.

Filters help it easy to select browser, OS and their respective resolutions.

You can easily access filters and manage your segments.

Segments here are the entities that you are selecting from the filters or the secondary searches from the iteration 1.

There are many more ways you can use to tackle up the problem. For the assignment, I have used only 2
