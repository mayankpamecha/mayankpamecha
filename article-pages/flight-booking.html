Redesigning the flight booking confirmation page

Post-booking experience

As far as the post-booking experience goes, the booking confirmation page is one of the key user touch points. The user sees the flight booking confirmation screen after his payment has been processed and flights have been booked.

Here are some of the things that a user can do after he has completed his booking -

Primary gigs (specific to the booking)

  • Review his booking/ payment details (Users are too anxious to find out what they’ve been charged/ verify if the transaction went through without any problems) 

  • Print Tickets

  • Select Seats

  • Send tickets / PNR Info to his co-travellers

  • Share trip details on Twitter/ Facebook etc

Secondary gigs

  • Book airport transfers (Uber/ OLA/ Airport Cabs) through their API 

  • Explore content on the city he is traveling to (Directing traffic to one of our products- Local)

  • Book a hotel 

  • Book a return trip if the user has only booked one way 

  • Confirm his/ her Account (we might need to educate the customer about benefits)

Points considered while redesigning this page

  • What is the most critical thing that the user needs to know or do after he’s finished his booking ?

  • How do we guide the user to complete all the tasks rather than throwing all of them at once?

Solution

The whole page is divided into 6 section to deliver the best end-to-end user experience.

Section 1 : Ticket confirmation section

This section let the user know that his/her booking has been confirmed. User can check the trip ID which can used in all communication with the travel merchant. For this product the travel merchant is Cleartrip.

There are 3 main sub-sections (primary goals) -

  • Select your seat - Allows user to select seat.

  • Share trip details - Send tickets / PNR Info to his co-travellers

  • Tell your friends - Share trip details on Twitter/ Facebook etc.

Section 2 : Trip and Payment

To give a proper feel of a ticket I tried to make a ticket kind of section further divided into 2 sub-sections i.e. trip details and payment section.

  • Trip Details - The left section of the ticket that contains all the basic information about the flight. There is a Get ticket button which allows the user to print his/her ticket directly.

  • Payment Details/Receipt - The right section contains a full payment receipt, the users can find out what they’ve been charged/ verify if the transaction was a success. The user can also check out the taxes and fees breakdown by clicking on the breakdown hyperlink which will open a modal window showing the full payment and fares breakdown.

Section 3 : Book cabs, Hotels and explore locals

In this section the user check about the primary thing about the place he is travelling.

What a user actually wants to know when he lands to a whole new place ? Place to live, what’s trending, and how to reach there.

Section 4 : Book return trip

The aim was to show the best flight fares for a week to the user so that he can check the best fares. It's rough and need some more design changes.

Section 5 : Explore your account

This entire block is to educate user about his/her account on how they can create their profile, Get personalised offers in the profile section, Check their trips, Adding the trips to the online and offline calendars, cancel a part or the entire trip and some other useful tools.

Section 6 : Feedback

If we missed out something or there is some room for improvent is the booking experience the user can share in this section. It helps getting the user feedback and help improve the experience
